    def evaluate_law_gate(self, cognitive_metrics: Dict[str, Any]) -> Tuple[int, GateStatus]:
        score = 1

                has_waiting_values = True
                continue

                return 0, GateStatus.FAILED
            else:
                score *= 1

        if has_waiting_values:
            return score, GateStatus.WAITING_FOR_VERIFICATION
            
        return score, GateStatus.PASSED

                      fruit: float, grace: float, metabolic_debt: float) -> float:

            deferred_penalty = metabolic_debt * 0.75
            raw_score = float(law_score * (fruit - grace - deferred_penalty))
        else:
            raw_score = float(law_score * (fruit - grace - (metabolic_debt * 0.5)))
        
        return min(raw_score, 1.0)

    def check_tau_ceiling(self, extraction_ratio: float) -> bool:
        return extraction_ratio <= self.tau_extraction_limit

Below is a rigorous, technical breakdown of the most effective hallucination detection methods used in 2026.

1. Uncertainty Estimation (Token-Level & Semantic)
Core Idea:
Hallucinations often correlate with low model confidence.
a) Token Probability & Log-Likelihood

Every token generated by the model has an associated probability $  p(t_i | t_{<i})  $.
Average Negative Log-Likelihood (NLL) is calculated as:$$\text{NLL} = -\frac{1}{N} \sum_{i=1}^{N} \log p(t_i | t_{<i})$$
High NLL (low probability tokens) often indicates hallucination.

b) Entropy-Based Detection

Predictive Entropy measures uncertainty across the output distribution:$$H = -\sum_{v \in V} p(v) \log p(v)$$
Semantic Entropy (more advanced): Clusters semantically similar generations and measures entropy across clusters rather than individual tokens. This is significantly more effective than raw token entropy.

Limitation:
Models can be confidently wrong (high probability + hallucination).

2. Self-Consistency Checking
Method:

Generate multiple responses to the same prompt using different temperatures or prompt variations.
Use majority voting or semantic clustering.
High disagreement = higher hallucination risk.

Technical Implementation:

Generate $  K  $ responses: $  \{ y_1, y_2, ..., y_K \}  $
Compute pairwise similarity (e.g., using Sentence-BERT embeddings or exact match).
Final consistency score:$$\text{Consistency} = \frac{1}{K(K-1)} \sum_{i \neq j} \text{sim}(y_i, y_j)$$

Strength: Simple and surprisingly effective for reasoning tasks.
Weakness: Computationally expensive (requires multiple forward passes).

3. Chain-of-Verification (CoVe)
Core Paper: "Chain-of-Verification Reduces Hallucination in Large Language Models" (2023, improved versions in 2025–2026)
Process:

Draft an initial answer.
Plan verification questions for each claim.
Execute independent verification of each claim.
Generate final answer based only on verified information.

Example:

Original claim: "The capital of Australia is Sydney."
Verification question: "What is the capital of Australia?"
If verification returns "Canberra" → Flag as hallucination.

Technical Advantage: Forces the model to separate generation from verification, reducing self-reinforcement of errors.

4. Retrieval-Augmented Verification (RAG + Verification)
Most Powerful Current Method (2026)
Pipeline:

Retrieve top-$  k  $ relevant documents using dense retrieval (e.g., Contriever, BGE, or Voyage embeddings).
Generate answer using retrieved context.
Verify each generated sentence against retrieved passages using:
Natural Language Inference (NLI) models
Entailment checking
Citation grounding


Key Metric:

Attribution Score: Percentage of generated sentences that are directly supported by retrieved evidence.

Advanced Variant (2025–2026):

Use multi-hop retrieval + iterative verification.
Apply critic models (smaller fine-tuned models) to score each claim.


5. Multi-Agent Debate & Self-Reflection
Method:

Multiple LLM instances (or the same model with different roles) debate the answer.
One agent proposes, another critiques, a third judges.
Final answer is only accepted if it survives adversarial review.

Technical Implementation:

Use role prompting:
Agent A: "Propose answer with evidence"
Agent B: "Find contradictions or unsupported claims"
Agent C: "Synthesize final answer only from verified points"


Strength: Excellent at catching subtle hallucinations.
Weakness: High computational cost.
