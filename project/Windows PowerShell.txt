## Windows-Specific Paste & Clipboard Problems (Very Common)
 
Windows users (especially on Windows Terminal, PowerShell, or portable/external drives like D:\) frequently report "I cannot paste" or broken clipboard behavior. This is one of the most painful friction points when using the TUI.
 
### Why Pasting Is Hard on Windows
- Windows Terminal + PowerShell often have inconsistent `Ctrl+V` behavior for images vs plain text.
- Many terminals swallow or transform clipboard contents.
- Running from a portable drive (D:\ etc.) can sometimes interact poorly with certain terminal features.
- Image paste normally requires `Alt+V` (Grok-specific workaround) instead of `Ctrl+V`.
 
### Recommended "I Cannot Paste" Workflow
When copy-paste feels broken, stop fighting it and use file-based capture instead. These commands let you share state with the agent with minimal friction:
 
**Best all-around: Start a full transcript**
```powershell
Start-Transcript -Path "D:\portable\terminal-output.txt"
# ... run your commands normally ...
Stop-Transcript
```
 
**Capture one command + see output live**
```powershell
Your-Command | Tee-Object -FilePath "D:\portable\output.txt"
```
 
**Windows Terminal users (easiest visual method)**
1. Right-click the tab at the top of Windows Terminal.
2. Choose **"Export text..."**.
3. Save to your portable drive (e.g. `D:\portable\terminal.txt`).
 
Then you can open the resulting `.txt` file and paste only the small relevant section (or just tell the agent the filename).
 
### Image Paste on Windows
- Screenshots / "Copy Image": Use **`Alt+V`** inside the Grok prompt (not `Ctrl+V`).
- Regular text/files: `Ctrl+V` usually works.
- If `Alt+V` also fails, fall back to the file-capture methods above.
 
### Portable Drive Tips
- Prefer consistent drive letters when possible (Disk Management → Change drive letter).
- When giving paths to Grok, use the actual current letter (e.g. `D:\portable\...`).
- Large models (if you later use local Aider + Ollama) benefit from fast external NVMe drives.
 
### Strong Recommendation for Heavy Users
If terminal integration pain (pasting, keybindings, detection) keeps blocking you, consider the escape hatch you already asked about:
 
**Switch to local Aider + Ollama** running directly against your `D:\portable` files. The local agent sees your files natively — no pasting required. This is often the highest-leverage long-term solution for portable Windows workflows.
 
See the Aider + Ollama setup guidance (or ask for the portable Windows recipe).
